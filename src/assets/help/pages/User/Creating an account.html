Creating an account

The only way for users to create accounts within the dashboard, is user needs to be first invited by dashboard's owner or admin.

  • Once dashboard's owner or admin sends an invite, user will receive an email.
  • By clicking a link in the email, user is redirected to create an account page.
  • On create an account page, user needs to provide:
    • Full name
    • Password
    • Address and Secret (optional)

If user already has address and secret, and if that address isn't already in use by another account within the dashboard, it can be used for creating a new account.

Otherwise, user gets newly generated address and secret, which for security reasons aren't stored anywhere, and should be saved in a safe place by a user.

User can login with email and password, without a need (but with an option) to login with address and secret.

How login works

Two options, login with:

  1. Email and password
  2. Address and secret
Address and secret are key-value pair generated and registered within a specific Hermes node, ie. Ambrosus Test Hermes (https://gateway-test.ambrosus.com). Having address and secret that are registered within Hermes, provides the holder with an ability to create/view certain amount of assets and events.

Login process:

  1. If using email and password, check whether account exists.
  2. If it does and if password is correct, browser receives encrypted token of users secret, and it's decrypted with provided password inside of the browser (client side), so secret is never sent over the internet, it always stays in it's raw form client side only.
  3. Furthermore, there's validation (server side), whether users account (address and secret) are registered within specific Hermes Node, if yes, user can login.
  4. Entire login process is very much secure.
