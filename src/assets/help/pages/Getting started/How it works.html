How it works

At the moment there is one centerlized Ambrosus dashboard.

There are 3 different deployment versions:

  1. Dev: for developing new app features.
  2. Test: for testing new features.
  3. Production: production version.

Dashboard is initially setup against a specific Ambrosus hermes node (ie. https://gateway-test.ambrosus.com), along with the owner of the dashboard.

To get a user account, dashboard's owner or admin need to send an invite to one or more users. Upon receiving an invite in the email, user can follow the link in the email and create an account.

With the new account, depending on role user has, he can create new assets, events, view all the assets created by him...
